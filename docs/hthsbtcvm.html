How to help the SBTCVM project, and what SBTCVM is.
Index


SBTCVM is quite the complex project.  A great way to get it moving along further is helping out development.  You don't even need to be an expert at programming or computers, simply testing code and helping with documentation, or simply spreading the word to interested people can help quite a lot. 


However if you are skilled in programming, (specifically python), and know about balanced ternary or are curious about it and willing to learn,  SBTCVM can use more development skills.  here is a overview of the kinds of help the SBTCVM project can use:

  • help with Documentation. users with basic knowledge of Balanced ternary are preferred.
  • help with Designing and improving SBTCVM's utilities and related tools.  You see, due to the sheer lack of tools related to balanced ternary,  the SBTCVM project has had to write many tools such as an assembler, launchers, and such, and many more utilities will likely need to be written as SBTCVM evolves, and grows more advanced.
  • help with designing and implementing SBTCVM's core architecture.  Due to the sheer lack of reference on balanced ternary computer design, trial and error, and research, are expected. 
  • SBTCVM is a ways away from running a practical, useful, Operating system, but with enough work, perhaps SBTCVM will provide an example of that as well.
  • Due to SBTCVM's complexity, help in the overall design from knowledgeable people would be appreciated.
  • Software to run in SBTCVM would be easier to develop with a reasonably high level, portable, programming language designed for balanced ternary computers.

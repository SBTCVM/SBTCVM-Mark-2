Q8:
How does SBTCVM and its utilities preform balanced ternary integer mathematics?

 A:
SBTCVM uses a instruction level tactic towards its VM design. meaning it interprets the VM program code using high level logic, Thus avoiding slow, complex, circuit emulation.

That's why SBTCVM is able to get such high speed operation. its mimicking the functionality of a mythical balanced ternary microprocessor and accompanying computer that doesn't even exist as a circuit diagram at this point.

of course SBTCVM needs some special support code to do that. and one of the key libraries, in fact arguably the most important, libbaltcalc.

libbaltcalc of course, works by using high-speed balanced ternary to decimal conversion, and decimal to balanced ternary conversion, to preform mathematical operations in decimal, thus using python's existing mathematics routines to quickly preform operations. those conversion routines are also used quite heavily by SBTCVM.

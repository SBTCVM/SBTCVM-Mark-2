Overview:
This section of SBTCVM's Documentation describes how various possible future features should be implemented, and acts as a general description of implementation Goals for SBTCVM.

Goal Symbol Key: (combinations of some of these may occur)
  • red   "!" = higher priority
  • green "C" = goal for SBTCVM Mark 2.* series.
  • green "L" = longer term goal for beyond Mark 2.* series, but the Mark 2.* series may get some work towards this goal as well.
  • blue  "L" = longer term goal for beyond Mark 2.* series that is highly unlikely to be implemented in the Mark 2.* series for various reasons such as technical and design limitations.
  • blue  "?" = possibility for beyond Mark 2.* series
  • green "?" = possibility for Mark 2.* series

Main Menu & Pause Menu:

  • Add a balanced ternary calculator tool to the main and pause menus. C

Internal SBTCVM balanced ternary programs, SBTCVM codebase, and related:
  • Design and develop a balanced ternary operating system. L?
  • Implement a "real" BIOS into SBTCVM's design. Rework the more "high level" VM instructions into it. (example: The TTY write port) L
  • Develop more interactive, complex, and interesting programs for SBTCVM to run. !
  • Improve SBTCVM's Design and code base. L

General Programming:

  • Look into developing an IDE and/or language "plugins" for various existing IDEs ??
  • Develop and implement a reasonably high level Balanced ternary Programming Language, complete with external interpreter and if possible a compiler for SBTCVM. Should not be too difficult to learn. !
